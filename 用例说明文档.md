``# 在线拍卖系统 — 用例说明文档

本文档按角色列出每个角色的用例，一个角色一个用例表。

---

## 一、普通用户

### 1.1 买家

| 项目 | 内容 |
|------|------|
| 用例名称 | 买家功能 |
| 用例标识号 | UC01 |
| 参与者 | 买家 |
| 用例描述 | 买家通过网页端或微信小程序端使用系统，可以浏览商品、参与竞拍、管理账户、使用社交功能和申请角色升级。 |
| 前置条件 | 用户已注册并登录系统（浏览商品和公告无需登录），账号未被禁用 |
| 后置条件 | 各项操作成功后系统更新对应数据 |
| 基本事件流 | 1. 商品浏览：买家进入首页查看轮播图和推荐商品；进入拍品列表页面按关键词搜索或按类目筛选商品；点击商品进入详情页查看图片轮播、视频、起拍价、当前最高出价、加价幅度、保证金金额、竞拍倒计时和出价记录列表；进入竞拍公告页面查看公告列表和详情；进入拍卖专场页面浏览专场列表和专场内商品。2. 竞拍参与：买家在商品详情页输入出价金额（须高于当前最高价至少一个加价幅度）参与竞拍；买家可设置代理出价上限，由系统在竞拍过程中自动代为出价；竞拍成功后买家在订单页支付尾款；买家在历史竞拍页面查看自己的出价记录和成交记录。3. 账户管理：买家在个人中心修改昵称、头像和密码；在保证金页面查看余额和交易明细，提交充值申请，申请保证金解冻；在订单页面查看待支付、已支付、已发货、已完成订单，对已完成订单申请发票。4. 社交互动：买家在留言板查看所有留言，发布新留言，编辑和删除自己的留言；在消息中心创建与客服的咨询会话，发送和接收消息，查看会话关联的订单信息。5. 角色升级：买家在申请成为卖家页面上传资质文件，填写申请信息，提交后由管理员审核。 |
| 扩展事件流 | 出价金额不足时提示须高于当前最高价至少一个加价幅度；买家不能竞拍本人发布的商品；竞拍时间窗口未到或已结束时系统拒绝出价；账号被禁用时无法登录；留言内容为空时拒绝提交；买家只能编辑和删除自己的留言；已有待审核申请时不可重复提交卖家申请。 |

### 1.2 卖家

| 项目 | 内容 |
|------|------|
| 用例名称 | 卖家功能 |
| 用例标识号 | UC02 |
| 参与者 | 卖家 |
| 用例描述 | 卖家由买家申请经管理员审核通过后产生。卖家继承买家的全部功能（浏览商品、参与竞拍、账户管理、社交互动），并新增商品管理和订单发货功能。 |
| 前置条件 | 用户已通过卖家资质审核并转为卖方角色，已登录系统 |
| 后置条件 | 各项操作成功后系统更新对应数据 |
| 基本事件流 | 1. 买家全部功能：卖家可浏览商品、参与竞拍、使用代理出价、支付尾款、管理保证金、查看订单、使用留言板和消息中心、查看历史竞拍记录。2. 商品管理：卖家在"我的商品"页面发布新商品，填写商品名称、选择类目、编辑描述，上传图片和视频，设定起拍价、加价幅度、保证金金额和竞拍起止时间，提交后商品进入待审核状态；卖家可编辑已发布商品的信息，修改商品描述、图片、价格等；卖家可下架自己发布的商品，下架后商品不再对买家可见。3. 订单发货：卖家对本人名下已支付尾款的订单执行发货操作，填写物流公司和物流单号。 |
| 扩展事件流 | 商品发布必填字段为空时提示具体缺失项；竞拍结束时间早于开始时间时系统拒绝；卖家只能操作自己发布的商品；卖家只能对本人名下的已支付订单发货；商品正在竞拍中时下架系统给出警告提示。 |

## 二、后台管理用户

### 2.1 管理员

| 项目 | 内容 |
|------|------|
| 用例名称 | 管理员功能 |
| 用例标识号 | UC03 |
| 参与者 | 管理员 |
| 用例描述 | 管理员负责系统的日常管理与审核工作，功能覆盖用户管理、商品管理、订单管理、拍卖管理、系统设置、留言板管理、工单复核和管理通知。管理员通过后台登录页面登录。 |
| 前置条件 | 管理员通过后台登录页面输入手机号和密码，系统校验角色为后台管理角色（3-10）后建立会话 |
| 后置条件 | 各项操作成功后系统更新对应数据，记录操作日志 |
| 基本事件流 | 1. 用户管理：管理员查看用户列表（按用户名、手机号、角色筛选），新增买方或卖方账号，编辑用户昵称、手机号和角色，启用或禁用用户账号。2. 卖家资质审核：管理员查看买家提交的资质申请，审核通过则移除买方角色并新增卖方角色，驳回则填写原因。3. 商品管理：管理员查看全部商品（按审核状态、商品状态、类目筛选），审核待上架商品（通过或驳回），编辑商品信息，上架或下架商品，设置竞拍起止时间，对违规或过期商品执行逻辑删除。4. 订单管理：管理员查看全部订单（按状态、买方、卖方筛选），编辑订单信息，更新支付和发货状态，处理退款申请。5. 拍卖管理：管理员执行落槌确认成交、拍卖延时、标记流拍、标记违约和标记异常出价记录；查看和审核历史竞拍记录，导出历史订单报表为Excel格式。6. 系统设置：管理员管理轮播图（新增、编辑、删除）和竞拍公告（新增、编辑、删除）。7. 留言板管理：管理员查看全部留言，回复留言，删除违规留言。8. 工单复核：管理员查看客服提交的工单，审核通过后工单关闭，不通过则填写原因退回客服重新处理。9. 管理通知：管理员进入通知管理页面，选择内部岗位接收人（角色5至10），编辑通知标题和正文后发送；查看已发送通知列表和接收明细。10. 个人管理中心：管理员修改个人基本信息和登录密码。 |
| 扩展事件流 | 手机号已注册时新增账号失败；管理员只能创建买方和卖方账号；管理员只能逻辑删除商品不能物理删除；管理员用户列表不包含后台管理角色用户；管理员只能查看自己发送的通知。 |

### 2.2 超级管理员

| 项目 | 内容 |
|------|------|
| 用例名称 | 超级管理员功能 |
| 用例标识号 | UC04 |
| 参与者 | 超级管理员 |
| 用例描述 | 超级管理员在数据库中权限分配与管理员的完全一致，功能差异全部由后端RoleCheckHelper控制。超级管理员拥有管理员的全部功能（含系统设置），并新增物理删除商品、创建全部角色账号、查看全部用户和查看全量通知四项能力。 |
| 前置条件 | 超级管理员已通过后台登录 |
| 后置条件 | 各项操作成功后系统更新对应数据 |
| 基本事件流 | 1. 管理员全部功能：超级管理员可使用管理员所拥有的全部功能（用户管理、资质审核、商品管理、订单管理、拍卖管理、系统设置、留言板管理、工单复核、管理通知、个人管理中心）。2. 物理删除商品：超级管理员在商品管理页面选择商品，点击物理删除按钮，系统弹出二次确认提示"数据不可恢复"，确认后系统从数据库中永久移除商品数据。3. 创建角色账号：超级管理员在新增用户时角色下拉框可见全部角色（1-10），可创建除超管外的所有角色账号。4. 查看全部用户：超级管理员的用户列表包含管理员在内的全部用户。5. 查看全量通知：超级管理员在通知管理页面可查看系统中所有管理员发送的通知。 |
| 扩展事件流 | 普通管理员界面不显示物理删除按钮，调用物理删除接口时系统拒绝执行；普通管理员新增用户时只显示买方和卖方选项。 |

### 2.3 拍卖师

| 项目 | 内容 |
|------|------|
| 用例名称 | 拍卖师功能 |
| 用例标识号 | UC05 |
| 参与者 | 拍卖师 |
| 用例描述 | 拍卖师负责拍卖流程的执行和商品审核工作，可以浏览前台商品、参与竞拍、审核商品、执行拍卖流程操作、查看订单和历史竞拍记录。 |
| 前置条件 | 拍卖师已通过后台登录 |
| 后置条件 | 各项操作成功后系统更新对应数据 |
| 基本事件流 | 1. 前台浏览与竞拍：拍卖师浏览前台商品列表和详情页，查看商品图片、视频、起拍价、最高出价等信息；拍卖师可参与竞拍，输入出价金额或设置代理出价上限。2. 商品审核：拍卖师在后台查看全部商品列表，审核待上架商品（通过或驳回并填写原因），编辑商品基本信息，对商品执行逻辑删除。3. 拍卖执行：拍卖师执行落槌确认成交、拍卖延时、标记流拍、标记违约和标记出价记录的异常类型。4. 订单与历史竞拍：拍卖师查看竞拍订单列表和详情，管理历史竞拍记录，查看成交明细，导出历史订单报表为Excel格式。5. 管理通知：拍卖师在管理沟通通知页面接收管理员发送的通知。6. 个人信息：拍卖师修改个人基本信息和登录密码。 |
| 扩展事件流 | 竞拍规则与买家相同：出价须高于当前最高价至少一个加价幅度，不可竞拍本人发布的商品；拍卖师不能执行物理删除商品；拍卖师不参与商品上下架操作。 |

### 2.4 客服

| 项目 | 内容 |
|------|------|
| 用例名称 | 客服功能 |
| 用例标识号 | UC06 |
| 参与者 | 客服 |
| 用例描述 | 客服是后台功能最少的角色，负责消息会话管理、留言板管理和工单处理。 |
| 前置条件 | 客服已通过后台登录 |
| 后置条件 | 各项操作成功后系统更新对应数据 |
| 基本事件流 | 1. 消息中心：客服进入消息中心查看所有咨询会话列表，进入会话后查看用户的消息并回复，查看与会话关联的订单信息了解用户咨询的具体商品和交易情况。2. 留言板管理：客服进入留言板管理页面查看全部留言（支持按发布者和时间筛选），回复用户在留言板中提出的问题。3. 工单处理：客服查看分配给自己的工单列表，分析问题并分类，填写处理意见或解决方案回复工单，问题解决后将工单提交给管理员复核。4. 管理通知：客服在管理沟通通知页面接收管理员发送的通知，查看通知内容。5. 个人信息：客服修改个人基本信息和登录密码。 |
| 扩展事件流 | 客服不能删除留言，只能回复；工单关闭后客服无法再修改；管理员复核不通过时工单退回客服重新处理。 |

### 2.5 财务

| 项目 | 内容 |
|------|------|
| 用例名称 | 财务功能 |
| 用例标识号 | UC07 |
| 参与者 | 财务 |
| 用例描述 | 财务是系统中唯一拥有资金管理权限的角色，保证金后台的全部功能由财务独占。财务的功能覆盖保证金管理、充值审核、佣金管理、发票管理、订单管理和历史竞拍管理。 |
| 前置条件 | 财务已通过后台登录 |
| 后置条件 | 各项操作成功后系统更新对应数据，记录操作日志 |
| 基本事件流 | 1. 保证金管理：财务进入保证金管理页面查看平台保证金汇总（总余额、冻结金额、可用金额）；查看每个用户的保证金余额和交易流水明细；手动为用户充值保证金（输入金额和备注后执行）；冻结用户保证金（输入金额和原因）；解冻用户保证金（输入原因）。2. 充值审核：财务在充值申请页面查看用户提交的充值申请列表，审核付款凭证，通过则为用户增加保证金余额，驳回则填写原因。3. 佣金管理：财务在佣金页面设置佣金费率（按成交价百分比），执行佣金结算（系统自动计算每笔成交订单的佣金并扣费），查看和导出扣费记录为Excel格式。4. 发票管理：财务查看用户发票申请列表（发票抬头、税号、订单信息），对符合条件的申请上传已开好的发票文件，对不符合条件的申请做驳回处理并填写原因。5. 订单管理：财务查看全部订单详情（按状态、买方、卖方筛选），编辑订单信息和支付状态，处理退款申请并执行退款。6. 历史竞拍管理：财务按买方条件查询竞拍记录，查看成交明细，导出历史订单报表为Excel格式。7. 管理通知：财务在管理沟通通知页面接收管理员发送的通知。8. 个人管理中心：财务修改个人基本信息和登录密码。 |
| 扩展事件流 | 充值金额为负数时系统拒绝；冻结金额超过用户可用余额时拒绝；佣金费率超合理范围时系统警告；退款金额超出订单金额时系统拒绝执行。 |

### 2.6 运营

| 项目 | 内容 |
|------|------|
| 用例名称 | 运营功能 |
| 用例标识号 | UC08 |
| 参与者 | 运营 |
| 用例描述 | 运营是后台功能最丰富的角色，负责类目管理、系统设置、商品管理、专场管理、限时拍管理、推荐位管理、数据看板和订单发货。 |
| 前置条件 | 运营已通过后台登录 |
| 后置条件 | 各项操作成功后系统更新对应数据 |
| 基本事件流 | 1. 商品浏览：运营浏览前台商品列表和商品详情页。2. 类目管理：运营新增一级、二级或三级类目，编辑类目名称，查看类目树结构。3. 系统设置：运营管理轮播图（新增并上传图片、设置跳转链接和排序、编辑、删除）和竞拍公告（新增、编辑标题和内容、删除）。4. 商品管理：运营在后台商品管理页面审核待上架商品（通过或驳回并填写原因），编辑商品信息，上架或下架商品，设置竞拍起止时间，对商品执行逻辑删除。5. 专场管理：运营新增拍卖专场（设置名称、封面图和活动时间），编辑专场信息，删除不再使用的专场，将商品加入或移出专场。6. 限时拍管理：运营在限时拍管理页面为指定商品设置竞拍起止时间，限时拍竞拍窗口短于常规竞拍。7. 推荐位管理：运营添加商品到首页推荐位，移除不再推荐的商品，调整推荐商品的展示顺序。8. 数据看板：运营查看平台运营统计数据（用户增长趋势、商品上架数量、成交金额走势、热门类目分布等）。9. 订单发货：运营查看订单列表和详情，对已支付订单执行发货操作（填写物流公司和物流单号）。10. 管理通知：运营在管理沟通通知页面接收管理员发送的通知。11. 个人信息：运营修改个人基本信息和登录密码。 |
| 扩展事件流 | 类目名称重复时提示；专场中有在拍商品时不可删除；竞拍结束时间早于开始时间时拒绝；推荐位达上限时提示；运营不能执行物理删除商品。 |

### 2.7 风控

| 项目 | 内容 |
|------|------|
| 用例名称 | 风控功能 |
| 用例标识号 | UC09 |
| 参与者 | 风控 |
| 用例描述 | 风控负责平台的交易风险监控，通过独立的路由 `/risk-dashboard` 访问风控中心。风控的功能包括异常出价检测、恶意参拍分析、风险等级管理和日志查看。 |
| 前置条件 | 风控已通过后台登录 |
| 后置条件 | 各项操作成功后系统更新对应数据 |
| 基本事件流 | 1. 异常出价检测：风控在风控中心查看系统自动检测到的异常出价记录，支持按商品维度和异常类型（疑似围标、恶意竞价、机器人出价等）筛选。2. 恶意参拍分析：风控查看恶意出价人汇总信息、围标嫌疑出价人列表及其关联出价记录，查看风险活跃用户列表（参与异常竞拍活动频率较高的用户）。3. 风险等级管理：风控在用户列表页面为每个用户设置风险等级（低风险、中风险、高风险）；风控在冻结或解封申请页面填写申请理由，以工单形式提交申请，流转至管理员复核，管理员复核通过后冻结或解封才实际执行。4. 日志与历史：风控查看系统操作日志了解用户行为轨迹，查看历史竞拍记录。5. 管理通知：风控在管理沟通通知页面接收管理员发送的通知。6. 个人中心：风控修改个人基本信息和登录密码。 |
| 扩展事件流 | 高风险用户参与竞拍时可能需要缴纳更高保证金；冻结或解封申请须经管理员复核通过后才执行；异常出价记录由系统自动分析生成，风控仅查看不创建。 |

### 2.8 审计

| 项目 | 内容 |
|------|------|
| 用例名称 | 审计功能 |
| 用例标识号 | UC10 |
| 参与者 | 审计 |
| 用例描述 | 审计负责系统操作的可追溯性审查，通过独立的审计中心页面查看三类日志。审计仅拥有只读权限，不拥有任何数据的写入、修改或删除权限。 |
| 前置条件 | 审计已通过后台登录 |
| 后置条件 | 日志数据以列表形式展示 |
| 基本事件流 | 1. 操作日志：审计进入审计中心的操索日志页面，查看系统的操作记录（操作时间、操作人、操作类型、操作对象和操作前后的数据变化）。2. 登录日志：审计查看用户的登录记录（登录时间、IP地址、登录方式为网页端或小程序端、登录结果为成功或失败）。3. 敏感操作审计：审计查看被标记为敏感操作的日志记录，包括用户删除、商品删除、保证金冻结、佣金结算等涉及安全或资金的关键操作。4. 管理通知：审计在管理沟通通知页面接收管理员发送的通知。5. 个人信息：审计修改个人基本信息和登录密码。 |
| 扩展事件流 | 审计仅可查看日志，不能修改或删除任何数据；日志数据量过大时系统分页展示；审计无权限访问日志查看之外的任何功能页面。 |

## 三、微信小程序端

| 项目 | 内容 |
|------|------|
| 用例名称 | 微信小程序端功能 |
| 用例标识号 | UC11 |
| 参与者 | 买家、卖家 |
| 用例描述 | 微信小程序端仅面向买方和卖方两种普通用户，定位为竞拍入口，不包含任何后台管理功能。用户通过微信授权登录后可使用首页浏览、专场查看、商品详情查看、竞拍出价、我的商品查看和个人中心功能。 |
| 前置条件 | 用户完成微信授权获取openid，输入手机号后系统自动检测并登录 |
| 后置条件 | 各项操作成功后系统更新对应数据 |
| 基本事件流 | 1. 登录：用户完成微信授权获取openid，输入手机号后系统自动检测是否已注册，未注册则自动创建买方账号，已注册则直接登录，系统下发JSESSIONID会话，登录后提示用户设置密码（可选）。2. 首页：首页展示竞拍中的商品列表（主图和当前最高出价），用户下拉刷新获取最新商品数据。3. 专场：用户切换到专场Tab页查看拍卖专场列表，点击专场查看专场内的商品。4. 商品详情：用户点击商品进入详情页，查看图片轮播、视频播放、起拍价、当前最高出价、加价幅度、竞拍倒计时和出价记录列表。5. 竞拍出价：用户在竞拍时间窗口内输入出价金额（须高于当前最高价至少一个加价幅度）参与竞拍，用户不能竞拍本人发布的商品。6. 我的商品：用户查看本人已竞得的商品列表（成交价、成交时间和订单状态）。7. 个人中心：用户查看微信头像、昵称、手机号和当前角色，设置登录密码（设置后可用同一账号登录网页端），买方角色用户可在页面中看到"申请成为卖家"入口。 |
| 扩展事件流 | 用户选择不设置密码则仅限小程序端使用，无法登录网页端；出价金额不足时提示须高于当前最高价至少一个加价幅度；用户是商品发布者时系统拒绝出价；竞拍时间未到或已结束时出价按钮不可用；网络异常时下拉刷新提示"网络异常，请稍后重试"。 |

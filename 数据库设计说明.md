# 数据库设计说明 - 用户角色多角色支持

## 一、问题分析

### 原有设计的问题：
1. `user_role` 字段是 `tinyint` 类型，只能存储单一角色值
2. 不支持"买方用户可以申请成为卖方用户"的业务需求
3. 缺少卖方资质证明材料字段

### 业务需求：
- 用户注册时可以选择"买方用户"或"卖方用户"
- **买方用户可以申请成为卖方用户**（需管理员审核）
- 审核通过后，用户同时拥有买方和卖方两种身份

## 二、数据库修改方案

### 1. 修改 `user_role` 字段类型
- **原类型**：`tinyint NOT NULL`
- **新类型**：`varchar(20) NOT NULL DEFAULT '1'`
- **说明**：支持多个角色，用逗号分隔
  - `"1"` = 只有买方身份
  - `"2"` = 只有卖方身份（注册时直接选择卖方）
  - `"1,2"` = 同时拥有买方和卖方身份（买方申请成为卖方后）
  - `"3"` = 管理员

### 2. 新增字段

#### `seller_certificate_files` (text)
- **说明**：卖方资质证明材料文件路径
- **格式**：多个文件用逗号分隔
- **示例**：`"/upload/certificates/2025/01/cert1.jpg,/upload/certificates/2025/01/cert2.pdf"`

#### `seller_audit_apply_time` (datetime)
- **说明**：买方用户提交申请成为卖方的时间

#### `seller_audit_time` (datetime)
- **说明**：管理员审核的时间

#### `seller_audit_user_id` (bigint)
- **说明**：执行审核操作的管理员ID

## 三、字段状态说明

### `seller_audit_status` 状态流转：
```
0 (未提交) 
  ↓ [买方用户提交申请]
1 (待审核)
  ↓ [管理员审核]
2 (审核通过) → user_role 更新为包含 "2"
3 (审核驳回) → 可以重新申请
```

### `user_role` 值变化：
- **注册为买方**：`user_role = "1"`
- **注册为卖方**：`user_role = "2"`，`seller_audit_status = 1`（需审核）
- **买方申请卖方**：`user_role = "1"`，`seller_audit_status = 1`（待审核）
- **审核通过后**：`user_role = "1,2"`，`seller_audit_status = 2`

已创建 `sql/update_user_table_for_multi_role.sql` 文件，包含：
1. 修改 `user_role` 字段类型
2. 添加新字段
3. 更新索引

## 五、代码修改

### 1. 实体类 (`AuctionUser.java`)
- `userRole` 字段类型从 `Integer` 改为 `String`
- 添加新字段的 getter/setter

### 2. Controller (`AuctionUserController.java`)
- 修改查询逻辑，支持多角色查询（使用 LIKE）
- 添加 `applySeller()` 接口：买方申请成为卖方
- 修改 `auditSeller()` 接口：审核通过后更新 `user_role` 为包含 "2"

### 3. 查询逻辑调整
- 查询角色时使用 `LIKE` 而不是 `=`，因为角色可能是 `"1,2"` 格式

## 六、使用示例

### 1. 买方用户申请成为卖方
```java
POST /api/OnlineAuction/auctionUser/{id}/apply-seller
参数：certificateFiles (文件路径，逗号分隔)
```

### 2. 管理员审核
```java
PUT /api/OnlineAuction/auctionUser/{id}/seller-audit
参数：
  - auditStatus: 2=通过, 3=驳回
  - auditRemark: 驳回原因（驳回时必填）
  - auditUserId: 审核人ID（可选）
```

### 3. 检查用户是否拥有某个角色
```java
// 检查是否是买方用户
boolean isBuyer = user.getUserRole().contains("1");

// 检查是否是卖方用户
boolean isSeller = user.getUserRole().contains("2");

// 检查是否是管理员
boolean isAdmin = user.getUserRole().contains("3");
```

## 七、注意事项

1. **角色值规范化**：创建用户时，会自动去重和排序角色值
2. **查询性能**：由于使用 `LIKE` 查询，建议在 `user_role` 字段上添加索引（如果需要）
3. **数据迁移**：现有数据的 `user_role` 值会自动转换为字符串格式（如 `1` → `"1"`）
4. **向后兼容**：如果现有代码使用 `Integer` 类型，需要更新为 `String` 类型
